# CodexTheme Skin-First SEO Design

## Goal

Capture the current high-intent `Codex skin` search vocabulary without changing the `CodexTheme` brand, domain, routes, gallery structure, or installation flow.

## Approved positioning

- Keep `CodexTheme` as the brand and `codextheme.tech` as the canonical domain.
- Use `Codex skins` as the home page's primary acquisition phrase because the available products include backgrounds, translucent interface treatments, coordinated colors, and complete Home and Session presentation.
- Retain `Codex themes` as the secondary category phrase so the page remains relevant to users searching for conventional appearance themes.
- Continue calling individual catalog records `themes` inside the product interface and existing `/themes/[slug]` routes. This patch changes search positioning, not the package model.

## Public copy

- Home title: `Codex Skins & Themes for Codex Desktop | CodexTheme`
- Home H1: `Codex skins that turn your workspace into a world.`
- Description: `Discover immersive Codex skins and themes with real Home and Session previews, then install a pinned release with one command.`
- Repository eyebrow: `CODEX DESKTOP / SKINS & THEMES`
- Structured-data alternate name: `Codex Skins & Themes`
- Social preview alt text: `CodexTheme — skins and themes for Codex Desktop`

The supporting lead must explain the visual scope naturally and mention both terms without keyword repetition.

## Boundaries

- No domain, route, sitemap, gallery-card, or package-format changes.
- No duplicate `/skins` landing page.
- No keyword stuffing or mechanical replacement of every `theme` label.
- Preserve GA4, canonical metadata, favicon, Open Graph image, and the pinned installation command.

## Verification

The rendered home-page test must assert the new title, H1, description, structured-data alternate name, repository eyebrow, and a natural `skins and themes` explanation. The production build and complete test suite must pass before publishing.

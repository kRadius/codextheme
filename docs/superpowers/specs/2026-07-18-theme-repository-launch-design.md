# codextheme.tech 主题仓库首发设计

## 决策摘要

codextheme.tech 的长期定位是持续增长的 Codex Desktop 主题仓库。首发不建设搜索、上传、账号或服务端平台；先用少量但差异明显的原创主题跑通最短闭环：

```text
浏览真实 Codex 效果 → 查看 Home / Session → 进入详情页 → 复制固定版本命令 → 应用或恢复
```

现有 `Midnight Circuit`、`Crimson Eclipse`、`Aurora Glass` 是同一视觉系列，只能算一个内容方向。当前迭代先保留三个可安装主题，并为六个视觉差异明显的后续主题建立完整数据、Gallery 和详情页骨架。图片、真实截图与安装包随后逐个补齐，不阻塞网站结构上线。

## 首发主题

新增主题使用原创生成素材，不使用动漫角色、游戏商标、名人或下载的第三方图片：

1. `ink-mountain` / 水墨山河：留白、雾山、墨色与朱砂。
2. `pixel-terminal` / 像素终端：复古像素城市、CRT 绿与琥珀。
3. `neon-tokyo` / 霓虹夜行：雨夜都市、蓝紫与桃红灯光，不出现品牌文字。
4. `obsidian-gold` / 黑金编辑部：黑色纸张、金属金线和克制的编辑感。
5. `sakura-observatory` / 樱花观测站：明亮樱花、玻璃穹顶与低饱和粉青色。
6. `abyss-station` / 深海工作舱：深海观察窗、生物冷光与海军蓝。

原有三个主题保留为 `Circuit Series`，状态为 `available`；新增六个主题状态为 `coming-soon`。CLI 只发现并打包 `available` 主题，网站可以渲染全部主题。后续主题只有在背景、真实截图、manifest、打包和真机验证全部通过后，才能改为 `available`。

## 真实预览规则

Gallery 和详情页不得把背景壁纸称为 Codex 预览。数据模型为每个主题预留：

- `wallpaper-home.jpg` 与 `wallpaper-session.jpg`：主题包内部背景素材。
- `preview-home.png` 与 `preview-session.png`：主题真实应用到 Codex Desktop 后的截图。

截图使用统一窗口尺寸和同一台 macOS 设备。Home 截图保留原生首页、项目选择器、建议区和输入框；Session 截图使用不包含私人数据的固定演示任务，并展示正文、代码块、工具输出和输入框。截图发布前检查用户名、项目路径、任务名和私人消息。

真实截图字段允许为空。为空时网站不得使用壁纸或 CSS 拼装图冒充真实效果，而是显示统一的 `真实截图待补齐` 空状态。`coming-soon` 主题不显示复制命令，详情页明确显示 `完成验证后开放安装`。

## 首页信息架构

首页是小型仓库，不是品牌落地页，也不是主题编辑器：

1. 紧凑导航：品牌、全部主题、安装帮助、提交主题、GitHub。
2. 紧凑首屏：`为 Codex 找一套新界面`、真实主题数量、一个滚动到 Gallery 的主操作。
3. Gallery：三列卡片，首屏能看到第一行；每张卡片使用真实 Codex Home 截图，可切换 Session。
4. 卡片元数据：中文名、英文名、系列/风格标签、验证状态、兼容平台。
5. 主题详情：两张真实截图、说明、兼容范围、唯一复制命令、恢复说明和相似主题。
6. 轻量投稿入口：链接到 GitHub Issue，不建设账号、审核后台或上传服务。
7. 紧凑安全说明与页脚。

“精选”和“验证”只作为质量标签，不再使用“不是主题仓库”“我们替你做过选择”“为什么只有三个主题”等叙事。

## 数据和扩展边界

静态 catalog 为未来搜索保留字段：`slug`、中英文名、描述、系列、标签、模式、accent、surface、作者、兼容版本、更新时间、壁纸路径、真实预览路径和命令。首页第一版只读取并渲染，不实现搜索、筛选或排序 UI。

主题达到 12 个后可以加入纯前端搜索与标签筛选；不需要数据库。用户上传只有在能够完成“上传 → 真实预览 → 生成主题 → 应用”的完整闭环时才进入范围。

## 视觉语言

网站采用冷灰纸张、墨黑、索引蓝和证明绿，接近实体样本目录而不是黑紫 AI 落地页。大胆视觉只花在真实 Codex 截图上；页面装饰保持平、硬、可扫描。

- 标题：窄体无衬线中文/英文组合。
- 数据和标签：等宽字体。
- 卡片：白纸底、细黑边、轻微实体投影。
- 状态：蓝色用于主操作，绿色只用于已验证。
- 动效：截图 Home/Session 交叉淡入；尊重 reduced-motion。

## 明确不做

- 搜索框、筛选器、热门排行或虚假下载量。
- 图片上传、在线编辑器、AI 生图入口。
- 登录、收藏、评论、支付、数据库或对象存储。
- 未完成安装闭环的“预览一下”功能。
- 使用壁纸、合成窗口或插画替代真实 Codex 截图。

## 验收标准

1. 网站 catalog 包含九个唯一主题；CLI 只识别三个 `available` slug。
2. 新增六个主题拥有完整元数据、明确的 `coming-soon` 状态和可后补资源路径，不要求本轮生成背景或安装包。
3. 每个 Gallery 卡片使用真实 Codex 截图或明确的空状态，不得误导。
4. 首页第一屏能看见 Gallery，且不出现搜索或上传入口。
5. 每个主题详情页只有一个复制应用命令主按钮。
6. 桌面与移动布局可用，键盘焦点清晰，reduced-motion 有效。
7. 自动测试、类型检查与 Next.js 构建通过后可以部署结构预览；只有标记 `available` 的主题需要通过主题打包门槛。
